================================================================================
          РАСШИРЕННОЕ РУКОВОДСТВО ПО ДОПОЛНИТЕЛЬНЫМ ФЛАГАМ УТИЛИТЫ
================================================================================

ПОДКОМАНДА: diff (Сравнение файлов, каталогов и бинарных данных)
--------------------------------------------------------------------------------
-B / --binary               : Переключает движок в высокоскоростной бинарный режим.
                              Выполняет побайтовый LCP-анализ (Longest Common Prefix)
                              исполняемых файлов (.exe, .dll) или прошивок (.bin), 
                              упаковывая дельту во встроенный LZ4-контейнер.

--strip-trailing-cr         : Позволяет полностью игнорировать символы возврата 
                              каретки (\r). Это критически важно в Windows-среде,
                              так как защищает сравнение файлов Windows (CRLF) 
                              и Linux (LF) от ложных срабатываний.

-L <LABEL> / --label        : Позволяет подменить имя файла в заголовках '---' 
                              и '+++' на произвольный текст. Крайне полезно при
                              интеграции утилиты во внешние системы (например, Git).

--tabsize <NUM>             : Задает кастомную ширину колонок для визуального 
                              вывода в две колонки (-y / --side-by-side). 
                              По умолчанию ширина составляет 35 символов.

-I <RE>                     : Игнорирует изменения, если ВСЕ измененные строки 
                              внутри текущего блока (hunk) соответствуют заданному
                              регулярному выражению (например, -I "version = ").

-X <PATTERN> / --exclude    : Задает маску исключений при рекурсивном обходе папок.
                              Файлы, соответствующие регулярному выражению 
                              (например, .*\.log), будут полностью проигнорированы.

--speed-large-files         : Включает внутренние эвристические оптимизации проходов
                              для ускорения сверки текстовых файлов огромного размера.

-c / --context              : Переключает вывод в контекстный формат с фиксированным
                              числом строк окружения сверху и снизу изменений.

-y / --side-by-side         : Переключает вывод в формат двух колонок для наглядного
                              сопоставления различий на экране терминала.

-e / --ed                   : Форматирует вывод в виде скрипта команд для текстового
                              редактора 'ed'.

-i / --ignore-case          : Включает регистронезависимое сравнение содержимого.

-B / --ignore-blank-lines   : Пропускает изменения, если они заключаются исключительно
                              в добавлении или удалении пустых строк.


ПОДКОМАНДА: patch (Применение текстовых и бинарных LZ4 патчей)
--------------------------------------------------------------------------------
-B / --binary               : Переключает аппликатор патчей в бинарный режим. 
                              Парсит сжатый контейнер WIN_BIN, запускает потоковый
                              декодер LZ4 в памяти и собирает измененный файл или
                              прошивку на накопителе на максимальной скорости.

-p <NUM> / --strip          : Задает количество вложенных папок, которые необходимо
                              отсечь из путей к файлам, указанных внутри файла патча
                              (например, превращает a/src/main.rs в src/main.rs при -p1).

-F <NUM> / --fuzz           : Задает максимальный фактор размытия (по умолчанию: 2).
                              Позволяет накладывать патч, даже если номера строк в 
                              файле сместились вверх или вниз относительно оригинала.

-R / --reverse              : Автоматически инвертирует знаки патча. Меняет операции
                              добавления (+) и удаления (-) местами, выполняя чистый
                              откат ранее примененных изменений.

-r <FILE> / --reject-file   : Перенаправляет все сбои наложения (failed hunks) в 
                              один указанный глобальный файл отклонений вместо 
                              создания множества локальных файлов с расширением .rej.

--dry-run                   : Запускает симуляцию применения патча в памяти. Проверяет
                              целостность структуры и контекст без физической записи
                              изменений на жесткий диск.
